# НИИЧАЗ — Внутренняя информационная система

## Структура

```
niichaz-final/
├── index.html          ← Главная страница
├── build.sh            ← Скрипт сборки с обфускацией
├── css/
│   └── style.css
└── js/
    ├── data.js         ← Артефакты и мутанты (редактируй здесь)
    ├── auth.js         ← Список сотрудников (редактируй здесь)
    ├── supabase.js     ← Подключение к базе данных
    └── app.js          ← Логика приложения
```

---

## Первый запуск

### 1. Настроить домен в index.html

Открой `index.html`, найди массив `allowed` и впиши свой домен:

```js
const allowed = ["твой-логин.github.io", "localhost", "127.0.0.1"];
```

### 2. Добавить сотрудников

Открой `js/auth.js`, найди массив `USERS` и добавь записи:

```js
{
  id: "004",
  name: "Иванов Иван Иванович",
  login: "ivanov",
  password: "секретный_пароль",
  role: "Научный сотрудник",
  level: "junior",       // junior / senior / admin
  department: "Биология"
}
```

### 3. Разместить на GitHub Pages

1. Создай репозиторий на GitHub
2. Загрузи все файлы
3. Settings → Pages → Branch: main → Save
4. Готово: `твой-логин.github.io/название-репо`


## Управление базой данных

Реестр сталкеров, отчёты и цены хранятся в Supabase.
Управление через панель: **supabase.com → твой проект → Table Editor**

---

## Уровни доступа

| Уровень | Возможности |
|---------|-------------|
| `junior` | Просмотр всех разделов, написание отчётов |
| `senior` | + Редактирование цен артефактов и мутантов |
| `admin`  | Полный доступ + административная панель |
